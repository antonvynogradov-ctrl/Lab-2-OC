## Теоретичний базис та термінологія

### 1. Глосарій ключових понять (CLI & System)

| Термін | Значення | Функціональна роль |
|:-------|:---------|:-------------------|
| **Command** | Директива | Вказівка процесору на виконання конкретної операції. |
| **CLI** | Текстовий інтерфейс | Метод прямого керування ядром системи через консоль. |
| **Shell** | Інтерпретатор | Прошарок (напр., Bash), що перекладає команди користувача на мову ОС. |
| **Path (Absolute/Relative)** | Маршрут | Шлях до об'єкта від кореня (`/`) або від поточної позиції. |
| **Root** | Суперкористувач | Обліковий запис з необмеженими правами адміністрування. |
| **Permissions** | Матриця доступу | Сукупність прав (читання, запис, виконання) для суб'єктів системи. |

---

### 2. Принципи взаємодії з ОС

* **Консольний режим (CLI):** Робота в середовищі без графічної складової, де ресурси витрачаються лише на обробку команд.
* **Емулятор терміналу:** Вікно всередині GUI (наприклад, GNOME Terminal), що імітує роботу консолі для зручності розробника.
* **Віртуальна консоль (TTY):** Автономний текстовий інтерфейс, доступний за межами графічної сесії.

---

### 3. Класифікація серверного ПЗ в Linux

Для забезпечення мережевих сервісів найчастіше використовуються:
1.  **СУБД:** PostgreSQL, MariaDB (структуровані дані).
2.  **Поштові стеки:** Postfix, Dovecot (обробка протоколів SMTP/IMAP).
3.  **Мережеві сховища:** NFS, Samba (спільний доступ до файлів у гетерогенних мережах).

---

### 4. Керування програмним забезпеченням

**Менеджер пакетів** — це системна утиліта для автоматизації встановлення та розв'язання залежностей ПЗ. 
* В екосистемі Debian/Ubuntu стандартом є **APT**. 
* Для RedHat-подібних систем використовується **DNF** або **YUM**.
* Сучасні універсальні формати: **Snap** та **Flatpak**.

---

### 5. Системна безпека та віртуалізація

**Захист даних** у Linux базується на декількох рівнях:
* Дискреційне керування доступом (права файлів).
* Мережеві екрани (Netfilter/UFW).
* Мандатне керування доступом (AppArmor/SELinux).

**Віртуалізація та контейнери:** Дозволяють ізолювати процеси (Docker) або створювати повні копії апаратних середовищ (VirtualBox) для безпечного тестування та масштабування інфраструктури.

---

### 6. Робота з терміналами (TTY)

В ОС Linux передбачено стандартний набір із **6 віртуальних консолей**:
* Перехід до текстового режиму: `Ctrl + Alt + F2–F6`.
* Повернення до графічного стека (X11/Wayland): зазвичай `Ctrl + Alt + F1` або `F7`.
* Система підтримує **мультисесійність**: один користувач може бути одночасно авторизований у різних терміналах та через SSH.
